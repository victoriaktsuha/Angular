What is Angular?

Angular is a Javascript Framework which allows you to create reactive Single-Page-Applications (SPAs)

A SPA is a single HTML file that render the changes just in the browser (DOM), creating a fast and reactive experience for not having to reach a server for each change of each page and each data. It allows you to create web applications that look and feel almost like mobile app, very fast. The server data is loaded in the background.



Project Setup and First App

1. Install latest version of Node.JS

https://nodejs.org/

2. Install Angular

You can install it globally, on your machine system, openning terminal or cmd from the start or locally, using a IDE, inside your project folder path.

> npm install -g @angular/cli@latest (@latest is optional, you can use a certain version e.g. @4.2.6)

3. Creating your Angular project

First, navigate to the folder you want to create the project inside, if you're not already; e.g. cd folder/folder_name

Then run:
> ng new project-name --no-strict

Choose config routing now or later: > ? Would you like to add Angular routing? (y/N)

Choose the stylesheet format (it can be change later on angular.json):> ? Which stylesheet format would you like to use ?

After finished, navigate to the project folder that you just created and run > ng serve and access the localhost address to see the initial page of your app



CLI Deep Dive & Troubleshooting

You encountered issues during the installation of the CLI or setup of a new Angular project?

A lot of problems are solved by making sure you're using the latest version of NodeJS, npm and the CLI itself.

Updating NodeJS:
Go to nodejs.org and download the latest version - uninstall (all) installed versions on your machine first.

Updating npm:
Run [sudo] npm install -g npm (sudo is only required on Mac/ Linux)

Updating the CLI

> [sudo] npm uninstall -g angular-cli @angular/cli
> npm cache verify
> npm cache clean --force
> [sudo] npm install -g @angular/cli

Here are some common issues & solutions:

1. Creation of a new project takes forever (longer than 3 minutes)
That happens on Windows from time to time => Try running the command line as administrator

2. You get an EADDR error (Address already in use)
You might already have another ng serve process running - make sure to quit that or use ng serve --port ANOTHERPORT to serve your project on a new port

3. My changes are not reflected in the browser (App is not compiling)
Check if the window running ng serve displays an error. If that's not the case, make sure you're using the latest CLI version and try restarting your CLI

#2 The Basics


How an Angular App gets Loaded and Started

  • 'localhost:4200' is where the development server hosted by CLI will host the Angular app;

  • The 'single-page' displayed by server is the index.html, which has the 'app-root' tag inside, so it can show the 'app.component.html' content

  • The App component folder is a componet that CLI created for us, becaming the 'root' component, the one that will tie together our whole application at the end

  • 'main.ts' file is the first code that is executed, and between some imports we have 'platformBrowserDynamic().bootstrapModule(AppModule)', responsable for initialize bootstrap

  • Resuming: Angular start with the 'main.ts' with the Bootstrap Module which receives the App Module as argument > in the App Module we declare App Component in the bootstrap array, which means that when bootstrap initialize with the main.ts, it should build the whole application with the App Component > the Angular analyze the App Component config and find the 'app-root' selector > Now, Angular is capable to handle with 'app-root' in the index.html, inserting the App Component content in it
  • *Other components is not added into the bootstrap array or in the index.html file, just App Component


Components are important!

Components are a key-feature in Angular. You build your application by composing it from a couple of components, created by your own; The App Component will be where we'll nest later our other components.

Each component has its own tmeplate, styling and business logic. It allows you to split up your complex application/web page into reusable parts without having to fit everything in one single template, one single script, and turn it easy to update and change



Creating a new component

You can create it manually, by adding a folder and its components files or by running ng new component name-component

Don't forget to add your component to app.module.ts, if you created it manually



Understanding the Role of AppModule and Component Declaration

Angular uses compnents to build web pages and uses modules to bundle different pieces, like components of your application, into packages, giving information do Angular of which features this application have and use.

In module, the decorator is @NgModule instead @Component and we import 'NgModule' instead 'Component'

@NgModule has 4 properties: declarations (receives component that is part of this module), imports (receives other modules), providers (reserved to services), bootstrap (receives the component which will start with the application)

Each new component added to declarations need to be imported
